# Security Policy

## Reporting a Vulnerability

Please do not open public issues for confirmed vulnerabilities.

Email the maintainers or use GitHub private vulnerability reporting once the public repository is created. Include:

- affected version or commit
- reproduction steps
- expected impact
- suggested fix, if known

We aim to acknowledge reports within 72 hours.

## Supported Versions

Until the first stable release, security fixes target the latest commit on `main`.
